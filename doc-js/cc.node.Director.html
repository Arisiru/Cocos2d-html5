Class: Director

cc.node. Director

A Director object is the root node of a game.
  • As the primary component, it is a glue which puts together platform features such as Input routing or Texture and image caching, and Scenes and game logic.
  • Each Director has a renderer, which acts on its own Canvas Object. Since V4, Cocos2D HTML5 allows for multiple
  • Director instances and each one can have a different renderer type.
  • Every Director present in a Document, will share a common Texture and Image cache for better resource management.
  • A Director object runs Scenes. The process of switching scenes is handled using a Transition object.
  • The preferred way of building scenes is by calling director.createScene() -> Scene

    enumerateChildren(patternName, callback)

    Enumerate al children of a node that matches a pattern. If a pattern starts with // the search will be recursively performed from the root node. It is only legal to define // at the beginning of the pattern. If a pattern starts with / the search will be performed from the root node. The pattern accepts the wildcard symbol '*' meaning any value will match. The pattern accepts the symbol '..' meaning it references a node's parent. Example patterns:
  • //* . This pattern will get all descendant nodes from a node.
  • /child0/grandchild1 . This pattern will get all grandchildren of a node with name grandchild1 that have a parent node with name child0.
  • /*\/grandchild0 . This pattern will get all grandchildren of a node which have the name grandchild0.
    • Parameters:
    Name Type Description
    patternName string a search pattern. Patterns are composed of regular expressions separated by slash / characters.
    callback EnumerateCallback a callback function invoked for each node that matches the pattern.
